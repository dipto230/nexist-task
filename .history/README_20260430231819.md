# Nexist Fullstack Support Engineer Assessment

## Project Overview

This project was developed as part of the technical assessment for the **Fullstack Support Engineer** role at **Nexist**.

The application is a simple ecommerce-style cart system built with **Next.js, TypeScript, Redux Toolkit, Express.js, Prisma, and PostgreSQL**.

The main goal of this project was to solve real-world support issues such as:

- Cart items disappearing after page refresh
- Duplicate products being added to cart
- Improving user experience for non-technical users

## Core Problems Solved

### 1. Cart State Persistence

Cart items remain saved after page refresh using `localStorage`.

### 2. Duplicate Product Prevention

Same product cannot be added multiple times.

Implemented in:

- Frontend UI check
- Redux state logic
- Backend validation

## User Experience Improvements

- Empty cart message
- Success/error toast notifications
- Disabled button for already added products
- Smooth animations using GSAP
- Responsive and clean design

## End-to-End Testing

Application was tested like a real user.

### Issue 1:
Cart data disappeared after refresh.

**Solution:** Saved and loaded cart data from `localStorage`.

### Issue 2:
Duplicate items could be added.

**Solution:** Product existence check before adding.

### Issue 3:
No action feedback.

**Solution:** Toast notifications added.
